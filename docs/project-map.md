# Project Map

This portfolio is organized around one engineering story: reliable applied AI systems.
The projects are not meant to be a pile of disconnected demos. Each one covers a part of
the path from retrieval and evaluation to serving, release gates, and classic MLOps.

## First-Screen Order

| Order | Project | Main Signal |
| ---: | --- | --- |
| 1 | Enterprise RAG Reliability Platform | RAG reliability, cited answers, evals, traces, provider comparison |
| 2 | Applied AI Eval Lab | Document intelligence, release-gate metrics, live static demo |
| 3 | RAG Forge | Retrieval benchmarking and regression gates |
| 4 | StreamInfer | Inference-serving mechanics, batching, backpressure, benchmark gates |
| 5 | MLGuard | Pre-deployment drift, regression, latency release gate |
| 6 | MLOps End-to-End Pipeline | Training, serving, monitoring, Docker-oriented MLOps basics |

## What The Portfolio Proves

- I can build local-first AI/ML systems with tests, docs, and runnable commands.
- I understand why evidence, evals, traces, and release gates matter after a demo works.
- I can explain tradeoffs in retrieval, serving, and model-release workflows.
- I can keep claims tied to proof artifacts instead of vague portfolio language.

## What It Does Not Claim

- It does not claim production users or external adoption.
- It does not claim that local benchmark numbers transfer to every machine.
- It does not claim hosted CI is the source of truth while local verification is being
  used.
- It does not claim Prism CLI is production-ready; Prism stays supporting/experimental.
